# 完整项目流程——Claude Code Skill

## 这是什么？

你有没有遇到过这样的情况——让 AI 写一个项目，它直接甩给你一个 HTML 文件就说"做完了"？

这个 Skill 就是为了解决这个问题而生的。它强制执行完整的工程化流程，确保 AI 在开始编码之前，先完成需求调研、方案设计、数据库建模和接口定义。**单 HTML 文件不算完整项目**，这是本 Skill 的核心底线。

## 核心特性

### 📏 智能规模判定
收到项目请求后，自动判断规模，适配不同流程强度：
- 🟢 **小型项目**（工具脚本、静态页面）→ 快速模式 4 步
- 🟡 **中型项目**（前后端分离、多模块）→ 标准模式 8 步
- 🔴 **大型项目**（微服务、高并发）→ 完整模式 12 步

### 🔗 跨阶段 ID 追踪
从需求到交付，所有产出物用统一 ID 串联（REQ → API → DB → MOD → TEST），后期验收或变更时不迷路，减少返工。

### 🛑 阶段门控机制
每个阶段完成后必须等用户确认才能进入下一阶段，不会出现"AI 自说自话写完所有代码"的情况。跳过必做阶段会触发风险评估提示。

### 🔍 自动检查点
每阶段结束时自动校验：PRD 的验收标准是否覆盖所有需求？接口是否都定义了错误码？数据库是否配置了索引策略？代码里有没有硬编码密钥？

## 安装方式

将本仓库克隆到 `~/.claude/skills/` 目录下即可：

```bash
git clone https://github.com/h1h1333740-max/full-project-workflow.git ~/.claude/skills/完整项目流程
```

重启 Claude Code 后，说"帮我开发一个 XX 系统"即可触发。

## 适用场景

| 你的需求 | 触发方式 |
|---------|---------|
| 从零开始新项目 | "帮我做一个在线商城" |
| 增量功能开发 | "加一个搜索功能"（自动走精简流程）|

## 流程总览

```
📋需求调研 → 📝PRD → 🎨原型 → 🔧技术选型 → 🏗️架构
    ↓          ↓       ↓         ↓          ↓
🗄️数据库 → 📡接口文档 → 💻编码 → 🔍审查 → 🧪测试 → ✅交付
    ↓          ↓       ↓        ↓        ↓       🎉
```

## 为什么需要这个 Skill？

AI 写代码很快，但"快"不等于"好"。没有前期设计直接编码的代价是——需求理解偏差、接口对不上、数据结构不合理，最后返工的痛苦远超前面省下的时间。这个 Skill 把工程化流程固化为 AI 的行为纪律，**前面多花 30 分钟对齐，后面省 3 天返工**。
